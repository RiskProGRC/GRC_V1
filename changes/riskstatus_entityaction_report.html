Overview

When a user selects a department from the dropdown in riskstatus.php, an AJAX call fires to entityAction.php. That file runs 5 main queries (risk, KI, control, recommend, action) and then loops over each result set to render the HTML tables. The problem is that inside each loop, additional individual SELECT queries are fired to look up usernames, process names, and action names — one per row.

With a department that has 10 risks, 10 KIs, 10 controls, 10 recommendations, and 10 actions, the current code fires 5 + 10 + 10 + 20 + 10 + 10 = 65 queries. After this fix it will always be exactly 6 queries, regardless of row count.

The Problem — N+1 Queries Per Loop

Risk loop +1 query/row → SELECT * FROM users WHERE id='$uid'
KI loop +1 query/row → SELECT * FROM users WHERE id='$uid'
Control loop +2 queries/row → SELECT * FROM process + SELECT * FROM users
Recommend loop +1 query/row → SELECT * FROM action WHERE id='$aid'
Action loop +1 query/row → SELECT * FROM process WHERE process_id='$pid'

Change 1 — Risk Query

Lines 9–13 & 139–164
1

Add users JOIN to risk query; remove per-row SELECT

Performance Saves 1 query/row

The risk query does not JOIN the users table, so the loop fetches the reviewer name with a separate SELECT per row. Add the JOIN and select the name columns directly.

Before — Query (lines 9–13)
$rselect=mysqli_query($con,"SELECT risk.risk_id,risk.risk_name,risk.nominee,risk.cause,
  risk.rdate,riskcat.name,risk.reviewer
  FROM risk
  INNER JOIN riskcat ON risk.rcat=riskcat.riskcat_id
  WHERE dept='$deptid'");
After — Query
$rselect=mysqli_query($con,"SELECT risk.risk_id,risk.risk_name,risk.nominee,risk.cause,
  risk.rdate,riskcat.name,
  CONCAT(users.fname,' ',users.sname) AS reviewer_name
  FROM risk
  INNER JOIN riskcat ON risk.rcat=riskcat.riskcat_id
  INNER JOIN users ON risk.reviewer=users.id
  WHERE dept='$deptid'");
Before — Loop (lines 139–164)
while($rrisk=mysqli_fetch_assoc($rselect)){
    $uid=$rrisk['reviewer'];
    $select=mysqli_query($con,"SELECT * FROM users WHERE id='$uid'");
    $row=mysqli_fetch_assoc($select);
    $user=$row["fname"].'&nbsp;'.$row["sname"];
    // ... table row output ...
    <td><?=$user?></td>   // reviewer cell
}
After — Loop
while($rrisk=mysqli_fetch_assoc($rselect)){
    // no extra query needed
    <td><?=$rrisk['reviewer_name']?></td>   // reviewer cell
}

The KI loop looks up the owner's name via a separate SELECT per row. Add the JOIN to return the name directly.

Change 3 — Control Query

Lines 24–34 & 246–272
3

Add process & users JOINs to control query; remove 2 per-row SELECTs

Performance Saves 2 queries/row

The control loop is the worst offender — it fires two extra queries per row (one for process name, one for reviewer name). Both can be absorbed into the main query.

Before — Query (lines 24–34)
$cselect=mysqli_query($con,"SELECT control.risk,control.control,control.control_id,
  control.process_id,control.cstrength,control.ctype,control.reviewer,
  risk.risk_name,control_strength.cs_name,control_type.ct_name
  FROM control
  INNER JOIN risk ON control.risk=risk.risk_id
  INNER JOIN control_strength ON control.cstrength=control_strength.strength_id
  INNER JOIN control_type ON control.ctype=control_type.ctype_id
  WHERE dept_id='$deptid'
  ORDER BY control.risk ASC");
After — Query
$cselect=mysqli_query($con,"SELECT control.risk,control.control,control.control_id,
  risk.risk_name,control_strength.cs_name,control_type.ct_name,
  process.process_name,
  CONCAT(users.fname,' ',users.sname) AS reviewer_name
  FROM control
  INNER JOIN risk ON control.risk=risk.risk_id
  INNER JOIN control_strength ON control.cstrength=control_strength.strength_id
  INNER JOIN control_type ON control.ctype=control_type.ctype_id
  INNER JOIN process ON control.process_id=process.process_id
  INNER JOIN users ON control.reviewer=users.id
  WHERE dept_id='$deptid'
  ORDER BY control.risk ASC");
Before — Loop (lines 246–271)
while($control=mysqli_fetch_assoc($cselect)){
    $pid=$control['process_id'];
    $select=mysqli_query($con,"SELECT * FROM process WHERE process_id='$pid'");
    $row=mysqli_fetch_assoc($select);
    $process=$row["process_name"];
    $uid=$control['reviewer'];
    $select=mysqli_query($con,"SELECT * FROM users WHERE id='$uid'");
    $row=mysqli_fetch_assoc($select);
    $user=$row["fname"].'&nbsp;'.$row["sname"];
    <td><?=$process?></td>
    <td><?=$user?></td>
}
After — Loop
while($control=mysqli_fetch_assoc($cselect)){
    <td><?=$control['process_name']?></td>
    <td><?=$control['reviewer_name']?></td>
}

Change 4 — Recommend Query

Lines 37–43 & 302–322
4

Add action LEFT JOIN to recommend query; remove per-row SELECT

Performance Saves 1 query/row

The recommend loop fetches the linked action text with a separate SELECT per row. A LEFT JOIN (not INNER, since the action link may be empty) absorbs this.

Before — Query (lines 37–43)
$rmd=mysqli_query($con,"SELECT recommend.risk_id,recommend.id,recommend.mrc,recommend.armc,
  recommend.action,recommend.status,recommend.timeline,risk.risk_name
  FROM recommend
  INNER JOIN risk ON recommend.risk_id=risk.risk_id
  WHERE dept_id='$deptid'
  ORDER BY recommend.risk_id ASC");
After — Query
$rmd=mysqli_query($con,"SELECT recommend.risk_id,recommend.id,recommend.mrc,recommend.armc,
  recommend.status,recommend.timeline,risk.risk_name,
  action.action AS action_name
  FROM recommend
  INNER JOIN risk ON recommend.risk_id=risk.risk_id
  LEFT JOIN action ON recommend.action=action.id
  WHERE dept_id='$deptid'
  ORDER BY recommend.risk_id ASC");
Before — Loop (lines 302–322)
while($recommend=mysqli_fetch_assoc($rmd)){
    $aid=$recommend['action'];
    $select=mysqli_query($con,"SELECT * FROM action WHERE id='$aid'");
    $row=mysqli_fetch_assoc($select);
    $action=$row["action"];
    <td><?=substr($action,0,40)?></td>
}
After — Loop
while($recommend=mysqli_fetch_assoc($rmd)){
    <td><?=substr($recommend['action_name'],0,40).(strlen($recommend['action_name'])>40?'...':'')?></td>
}
