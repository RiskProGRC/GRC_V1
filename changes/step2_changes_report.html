What Was the Problem
Critical — SQL Injection

Unsanitized AJAX POST Data in SQL Queries

Both files received values from $_POST and placed them directly into SQL query strings with no validation or escaping. Since these files are called via AJAX, any user can send a crafted request to read, modify, or delete any data in the database.

Vulnerable Pattern
$did = $_POST['dept_id'];
$select = "SELECT process_id FROM dept_process WHERE dept_id='$did'"; // $did injected directly

Fix applied: Replaced with OOP mysqli prepared statements. The ? placeholder separates SQL structure from data — user input can never be interpreted as SQL code.

Changes Applied
SQL Injection

Fix 1 — Project/ajaxdata.php

✓ Fixed

File: Project/ajaxdata.php

Before
$con = new mysqli("localhost","root","","grc");
$did = $_POST['dept_id'];
if($did){
    $select = "SELECT process_id FROM dept_process WHERE dept_id='$did'";
    $result = $con->query($select) or die($con->error);
    while($row = $result->fetch_assoc()){
        $pid = $row['process_id'];
        $select = "SELECT * FROM process WHERE id='$pid'";
        $presult = $con->query($select);
        $pname = $presult->fetch_assoc();
        echo '<option value='.$row['process_id'].'>'.$pname['name'].'</option>';
    }
}
After
include_once './connection/connect.php';

$did = $_POST['dept_id'];

if($did){
    $stmt = $con->prepare("SELECT process_id FROM dept_process WHERE dept_id=?"); // prepare query, ? = safe placeholder
    $stmt->bind_param("i", $did);    // bind $did as integer to ?
    $stmt->execute();                // run the query
    $result = $stmt->get_result();   // get results
    $count = $result->num_rows;      // count rows

    if($count > 0){
        echo "<option value=''>SELECT Process</option>";
        while($row = $result->fetch_assoc()){
            $pid = $row['process_id'];

            $stmt2 = $con->prepare("SELECT * FROM process WHERE id=?"); // get process name by ID
            $stmt2->bind_param("i", $pid); // bind $pid as integer
            $stmt2->execute();             // run query
            $pname = $stmt2->get_result()->fetch_assoc(); // fetch row

            echo '<option value="'.$row['process_id'].'">'.$pname['name'].'</option>';
            $stmt2->close(); // close inner statement
        }
    } else {
        echo '<option>No state Found</option>';
    }

    $stmt->close(); // close outer statement
}
Also fixed: Removed standalone new mysqli() — now uses shared $con from connect.php.
