Part A — usersadd.php

Registration Form — Styling, Responsiveness & UX Improvements

13 improvements beyond accessibility — covering layout responsiveness, input UX, visual polish, and missing form features that would make the form production-ready.

🎨 5 styling fixes 📱 3 responsive fixes ☀ 5 UX additions
Styling Improvements
S1

Remove inline <style> — move to scoped CSS file or shared stylesheet

Styling

The form embeds a <style> block of ~56 lines directly in the page. This means styles cannot be cached, shared, or reused. The same pattern exists in profile.php with nearly identical CSS.

Fix: Extract these styles to assets/css/user-form.css and include it from header.php only on user-related pages, or add a $extraCss variable pattern so the header includes page-specific sheets.
S2

Left avatar column is decorative-only — wastes layout space

Styling

The col-md-3 left column contains only a static silhouette image with no functionality. On a create user form this column adds no value — the image cannot be changed and there is no name beneath it because the fields are on the right.

Current
<div class="col-md-3 border-right">          <!-- 25% of layout -->
    <img class="rounded-circle mt-5" width="150px" src="...silhouette...">
    <span class="font-weight-bold"></span>   <!-- empty -->
</div>
Improved — replace with upload placeholder
<div class="col-md-3 border-right text-center py-5">
    <div class="avatar-upload-box" id="avatar-preview">
        <img id="avatar-img" class="rounded-circle" width="130" src="...silhouette..."
             alt="Profile photo preview">
        <label for="avatar-file" class="btn btn-sm btn-outline-secondary mt-3">
            📷 Upload Photo
        </label>
        <input type="file" id="avatar-file" name="avatar" accept="image/*" class="d-none">
    </div>
</div>
S3

Password label uses inline red style — replace with Bootstrap text-danger

Styling

Line 147: <h5 style="color: red;">ENTER PASSWORD</h5> uses both an inline style and wraps a label inside an h5. Labels should not contain headings — heading tags are for document structure, not visual size.

Before
<label class="labels"><h5 style="color: red;">ENTER PASSWORD</h5></label>
After
<label class="labels fw-bold text-danger" for="password">Password <span aria-hidden="true">*</span></label>
S4

Add visible required-field indicator (* asterisk) on all mandatory fields

Styling

There is no visual signal that any field is mandatory. Users discover errors only after clicking submit. WCAG 3.3.2 (Labels or Instructions) requires that required fields are identified before submission.

Pattern to apply to every required label
<label class="labels" for="fname">
    Name <span class="text-danger" aria-hidden="true">*</span>
</label>
Add a legend above the form: "Fields marked * are required." so the asterisk meaning is explicit.
S5

Add consistent spacing between form groups using mb-3

Styling

Field groups use a mix of mt-2, mt-3, and no margin. Bootstrap's mb-3 on each field wrapper gives uniform vertical rhythm without custom margins.

Current — inconsistent
<div class="row mt-2"> ... </div>
<div class="row mt-3"> ... </div>
<div class="col-md-12"> ... </div>  <!-- no margin -->
After — uniform
<div class="mb-3">
    <label class="labels" for="fname">Name <span class="text-danger" aria-hidden="true">*</span></label>
    <input type="text" id="fname" class="form-control" name="fname" ...>
</div>
Responsiveness Improvements
R1

Three-column layout does not stack on small screens

Responsive

The form uses col-md-3, col-md-5, col-md-4. On screens below 768 px the columns stack but the avatar column (col-md-3) still appears first at full width, pushing the actual form fields below the fold.

Fix — reorder with order utilities on mobile
<div class="col-md-3 border-right order-md-1 order-2">  <!-- avatar last on mobile -->
<div class="col-md-5 border-right order-md-2 order-1">  <!-- fields first on mobile -->
<div class="col-md-4          order-md-3 order-3">  <!-- roles third -->
Also hide the avatar column entirely on mobile: add d-none d-md-block to the avatar column div — it provides no value on small screens.
R2

Name fields use col-md-6 but have no col-12 fallback

Responsive

The First Name / Last Name pair use col-md-6 inside an already-narrow col-md-5 parent. At the md breakpoint these become very narrow (~200 px each). On smaller screens they stack, but Bootstrap requires an explicit col-12 (or col-sm-12) fallback.

After
<div class="col-12 col-md-6 mb-3">  <!-- First name --></div>
<div class="col-12 col-md-6 mb-3">  <!-- Second name --></div>
R3

Submit button alignment breaks on mobile

Responsive

The "ADD USERS" button uses text-center but is inside a narrow column that may not centre correctly on all viewports. Make it full-width on mobile.

After
<div class="mt-4 d-grid d-md-block text-md-center">
    <button class="btn btn-primary adduser-button" type="button">
        ADD USER
    </button>
</div>
d-grid makes the button stretch full-width on mobile; d-md-block reverts it to auto-width on desktop.

U2

Password strength meter

UX

The admin sets the user's initial password with no guidance on strength. A simple progress bar that responds to keyup on the password field signals compliance with password policies.

Add below the password field
<div class="mt-1">
    <div id="pw-strength-bar" class="progress" style="height:5px;">
        <div id="pw-strength-fill" class="progress-bar" role="progressbar"
             style="width:0%"></div>
    </div>
    <small id="pw-strength-label" class="text-muted"></small>
</div>
// In usersadd.js
$('#password').on('keyup', function () {
    const val = $(this).val();
    let score = 0;
    if (val.length >= 8) score++;
    if (/[A-Z]/.test(val)) score++;
    if (/[0-9]/.test(val)) score++;
    if (/[^A-Za-z0-9]/.test(val)) score++;
    const labels = ['', 'Weak', 'Fair', 'Good', 'Strong'];
    const colors  = ['', 'bg-danger', 'bg-warning', 'bg-info', 'bg-success'];
    $('#pw-strength-fill').css('width', (score * 25) + '%')
        .removeClass().addClass('progress-bar ' + colors[score]);
    $('#pw-strength-label').text(labels[score]);
});
U3

Real-time username availability check

UX

Currently, a duplicate username error only surfaces after form submission. A blur AJAX ping to a lightweight check endpoint gives instant feedback and prevents a round-trip failure.

JS — fire on blur of the username field
$('#uname').on('blur', function () {
    const val = $(this).val().trim();
    if (!val) return;
    $.getJSON(window.GRC.base + 'Project/usernamecheck.php', { uname: val }, function (res) {
        if (res.taken) {
            $('#uname').addClass('is-invalid');
            $('#uname-feedback').text('Username already taken.');
        } else {
            $('#uname').removeClass('is-invalid').addClass('is-valid');
        }
    });
});
Requires a new usernamecheck.php endpoint: SELECT COUNT(*) FROM users WHERE username = ? — returns {"taken": true/false}.
U4

Client-side validation before AJAX submit

UX

The current usersaction.php handler replies with the generic string "FORM ERROR" for every missing field, without indicating which field failed. A brief client-side pre-flight check is faster and more user-friendly.

Pre-flight in usersadd.js — before the $.ajax call
function validateForm() {
    let valid = true;
    const required = ['fname', 'sname', 'uname', 'email', 'phone', 'password'];
    required.forEach(function (id) {
        const $el = $('#' + id);
        if (!$el.val().trim()) {
            $el.addClass('is-invalid');  valid = false;
        } else {
            $el.removeClass('is-invalid');
        }
    });
    return valid;
}

New Methods to Add to usersClass.php
// Toggle access (0 = suspended, 1 = active)
public function toggleAccess(string $uid): string {
    $stmt = $this->prepare("UPDATE users SET access = 1 - access WHERE id = ?");
    $stmt->bind_param('s', $uid);
    $stmt->execute();
    return $stmt->affected_rows > 0 ? "Access updated" : "No change";
}

// Soft delete — set deleted_at timestamp
public function deleteUser(string $uid): bool {
    $stmt = $this->prepare("UPDATE users SET deleted_at = NOW() WHERE id = ?");
    $stmt->bind_param('s', $uid);
    $stmt->execute();
    return $stmt->affected_rows > 0;
}

// Dashboard stats aggregate
public function getStats(): array {
    $row = $this->fetchOne('users', null, null,
        "SELECT COUNT(*) total,
                SUM(access=1) active,
                SUM(access=0) suspended,
                SUM(roles=1) admins
         FROM users WHERE deleted_at IS NULL");
    return $row ?? [];
}

// Username availability check
public function usernameExists(string $uname): bool {
    $row = $this->fetchOne('users', 'username', $uname);
    return $row !== null;
}

// Proxy to departmentClass for use in user forms
public function getDepartmentList(): array {
    require_once __DIR__ . '/../department/departmentClass.php';
    return (new departmentClass())->showDept();
}
