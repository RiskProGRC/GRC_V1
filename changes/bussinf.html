1. Security — Fix Immediately

HIGH XSS on All Table Outputs

Every echo that prints database values into the HTML tables is unescaped. If any field in the database contains a <script> tag or HTML entity, it will execute in the browser. This affects all tab panes.

Fix: Wrap every output value in htmlspecialchars($value, ENT_QUOTES, 'UTF-8') before printing.

HIGH No CSRF Token on Group Form

The Groups "Add" form at line 636 POSTs directly to group.php with no CSRF token. All other save/edit operations use AJAX (which can include a token in headers), but this one bypasses that protection entirely.

Fix: Generate a $_SESSION['csrf_token'] and embed it as a hidden field; verify it in group.php.

MEDIUM File Upload Has No Client-Side Type Guard

Logo upload inputs at lines 657 and 776 have no accept attribute, so the file picker shows all file types. Server-side validation is essential but the browser-level guard is also missing.

Fix: Add accept="image/png, image/jpeg, image/gif" to each file input and validate the MIME type and size on the server.

4. Broken HTML

MEDIUM Duplicate <tr> in Users Table

Lines 365–366 have two consecutive <tr> opening tags inside the users table loop. This creates a phantom blank row above every user row and produces invalid HTML.

Fix: Remove the extra <tr> on line 365.

MEDIUM Broken HTML Comment Syntax

Five comment tags use a single dash: <!-___... instead of <!--. Found at lines 37, 83, 95, 607, 2129. Invalid comments may cause surrounding markup to be parsed incorrectly in some browsers.

Fix: Change <!- to <!-- and ensure each has a closing -->.

MEDIUM Shared id="table1" Across Multiple Tab Panes

The ID table1 is applied to tables in the Groups, Company, Users, Impact, Likelihood, Risk Category, and Control Type panes (lines 155, 201, 347, 398, 440, 482, 563). The DataTable initialiser at line 2148 uses querySelector('#table1'), which returns only the first match. All other tables with that ID are never initialized as DataTables.

Fix: Give each table a unique ID (table-group, table-company, etc.) and update the DataTable init block accordingly.

LOW Group Table Header/Body Column Count Mismatch

The Groups table <thead> at line 156 declares 6 columns (Reference, Name, Website, Country, Objectives, Logo) but the <tbody> outputs 7 columns — the 6 data columns plus an Action column with no matching <th>.

Fix: Add an <th>Action</th> header to the Groups table thead.
