What Was the Problem
Redundancy

Hardcoded Credentials in Two Files + Double DB Connection

DB credentials were hardcoded separately in both connect.php and connectionClass.php, with multiple commented-out credential sets for different environments mixed in. Additionally, connect.php was opening two connections to the same database — $con and $db — on every page load.

connect.php — Before
$con = mysqli_connect("localhost","root","","grc");
//$con = mysqli_connect("localhost","phpmyadmin","r1skPr0grc","grc");
//$con = mysqli_connect("localhost","root","r1skPr0grc","grc_demo");
//$con = mysqli_connect("localhost","riskprog_admin","B0dabod@","riskprog_test");

$db = new mysqli("localhost","root","","grc");           // redundant second connection
//$db = new mysqli("localhost","riskprog_admin","B0dabod@","riskprog_test");
//$db = new mysqli("localhost","phpmyadmin","r1skPr0grc","grc");
Changes Applied
New File

Project/connection/config.php

✓ Created

Single file that holds all DB credentials. This is the only place credentials need to be updated when deploying to a different environment.

config.php
// Database configuration — update credentials here only
define('DB_HOST', 'localhost');
define('DB_USER', 'root');
define('DB_PASS', '');
define('DB_NAME', 'grc');
Note: Add config.php to .gitignore before deploying to prevent credentials being committed to source control.
