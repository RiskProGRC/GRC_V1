1. Responsive Layout Fix

Bug Fix col-1 Crushed All Tables on Non-Large Screens

The section wrapper had col-1 col-lg-12. On any screen below the Bootstrap lg breakpoint (992px), the column was only 8.3% wide — all tables were invisible or broken. Changed to col-12 so the card is always full width.

Before — 8% wide on tablet/mobile
<div class="col-1 col-lg-12">
After — full width on all screens
<div class="col-12">

3. DataTables — Null Guards

Fix Null Guards Added to All 5 DataTable Inits

Previously, all 5 DataTable initialisations ran unconditionally. If any table element was absent (e.g. tab not yet rendered), it would throw a JS error and break the others. Each init now checks the element exists first.

Before — throws if element missing
let dataTable1 = new simpleDatatables
    .DataTable(table1);
let dataTable2 = new simpleDatatables
    .DataTable(table2);
// ... same for table3, table4, table5
After — safe on every page load
if (table1) new simpleDatatables
    .DataTable(table1);
if (table2) new simpleDatatables
    .DataTable(table2);
// ... same for table3, table4, table5

4. Nav-Tabs Theme — Matching bussinf

Style Nav-Tabs Themed to Match bussinf Color Scheme

The default Bootstrap nav-tabs (white background, grey borders) were restyled to match the bussinf nav-pill color scheme. Tabs remain horizontal — only the visual appearance changes. The active indicator uses a bottom border (gold #ffc107) instead of the left border used in bussinf's vertical pills.
