What Was the Problem
Performance

jQuery Loaded Twice on Every Page

layout/header.php already loads jQuery 3.5.1 from Google CDN on every page. Despite this, 69 page files each contained their own local <script> tags loading jQuery and jQuery UI again from assets/vendors/jquery/.

Pattern removed from 69 files
<script src="../assets/vendors/jquery/jquery.min.js"></script>
<script src="../assets/vendors/jquery/jquery-ui.min.js"></script>

Two jQuery instances running simultaneously can cause event handlers to fire twice, AJAX callbacks to run multiple times, and unpredictable UI behaviour.

Changes Applied
Updated

layout/header.php — jquery-ui.min.js moved here

✓ Fixed

Since all page files relied on jQuery UI, it was moved to layout/header.php alongside the CDN jQuery so it loads once globally for every page.

Before
<script src="https://ajax.googleapis.com/ajax/libs/jquery/3.5.1/jquery.min.js"></script>
After
<script src="https://ajax.googleapis.com/ajax/libs/jquery/3.5.1/jquery.min.js"></script>
<script src="../assets/vendors/jquery/jquery-ui.min.js"></script>
Result: jQuery and jQuery UI each load exactly once per page, from a single place.
