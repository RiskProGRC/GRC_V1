✅ ARIA Changes Applied

HIGH ✓ Applied Auth Gate Added

Unauthenticated visitors are now redirected to login before any code runs.

Before — no session check
include_once './users/usersClass.php';
$profile = $usersclass->profile($uid);
After
if (session_status() === PHP_SESSION_NONE) { session_start(); }

/* ARIA Phase 3 — auth gate for view pages */
if (empty($_SESSION['uid'])) {
    header('Location: ../Project/login/login.php');
    exit;
}

HIGH ✓ Applied IDOR Eliminated — PRG Pattern (UID Hidden from URL)

$_GET['id'] is gone. Profile is now navigated via POST from userslist.php; the target UID is stored in $_SESSION['profile_uid'] and the browser redirects to clean profile.php with no parameters. Changing the URL does nothing. Non-admins are always locked to their own UID in session.

Before — ID visible and editable in URL
/* profile.php */
$uid = $_GET['id'] ?? '';

/* userslist.php */
<a href="profile.php?id=<?= $uid_row ?>">Profile</a>
After — ID never in URL
/* profile.php — PRG handler */
if ($_SERVER['REQUEST_METHOD'] === 'POST' && isset($_POST['id'])) {
    $posted = (int)$_POST['id'];
    $_SESSION['profile_uid'] = $isAdmin ? $posted : $sessionUid;
    header('Location: profile.php');
    exit;
}
$uid = isset($_SESSION['profile_uid']) ? (int)$_SESSION['profile_uid'] : $sessionUid;

/* defense-in-depth: even if session is tampered, non-admins cannot see another profile */
if ($uid !== $sessionUid && !$isAdmin) { $uid = $sessionUid; }

/* userslist.php — POST form instead of anchor */
<form method="post" action="profile.php" class="d-inline">
    <input type="hidden" name="id" value="<?= $uid_row ?>">
    <button type="submit" class="btn btn-sm btn-primary">👤 Profile</button>
</form>

LOW ⚠ Pending profile() Fetches Password Hash via SELECT *

usersClass::profile() calls fetchOne('users', 'id', $uid) which internally runs SELECT * FROM users. The upassword bcrypt hash and any other sensitive columns land in $profile and are available throughout the template — unnecessary exposure even if not echoed.

Fix — override profile() to use explicit columns (usersClass.php:25)
public function profile(string $uid): ?array {
    $stmt = $this->prepare(
        "SELECT id, fname, sname, gender, dept_id, email, phone, username, access, roles
         FROM users WHERE id = ? AND deleted_at IS NULL LIMIT 1"
    );
    $stmt->bind_param('s', $uid);
    $stmt->execute();
    return $stmt->get_result()->fetch_assoc() ?: null;
}
