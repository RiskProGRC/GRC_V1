formvalidation.js — How It Works

The utility attaches to the global window.GRC object (same namespace used by the rest of the project). It exposes one function:

// signature
GRC.validate(formId, rules) // returns true | false

// minimal rule
{ field: 'fname', message: 'First name is required.' }

// rule with custom format check
{ field: 'email', message: 'Enter a valid email address.',
  validator: function(v) { return /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(v); } }

Validation sequence

  1. Clears all is-invalid / is-valid / feedback text from the previous run.
  2. For each rule: trims the value and checks it is not empty.
  3. If a validator function is provided and the field is non-empty, runs it for format checking.
  4. On failure: adds is-invalid, writes rule.message into the nearest .invalid-feedback element, adds d-block to force display.
  5. On success: adds is-valid.
  6. Returns false if any rule failed — the calling code must abort submission.

Finding the feedback element

The utility handles both plain inputs and inputs wrapped in .input-group:

  • Tries $el.nextAll('.invalid-feedback').first() (direct sibling — all plain inputs).
  • Falls back to $el.closest('.input-group').nextAll('.invalid-feedback').first() (password field).

Before / After — usermanagement.js

Before
function validateAddForm() {
  var valid = true;
  ['fname','sname','uname',
   'email','phone','password']
  .forEach(function (id) {
    var $el = $('#' + id);
    if (!$el.val().trim()) {
      $el.addClass('is-invalid');
      valid = false;
    } else {
      $el.removeClass('is-invalid');
    }
  });
  return valid;
}

// called with:
if (!validateAddForm()) { ... }
After
var addUserRules = [

Using GRC.validate() on Any Future Form

Include formvalidation.js on the page (before any script that calls it), then define a rules array and call GRC.validate() in your submit handler.

<!-- 1. Load the utility before your feature JS -->
<script src="../assets/js/formvalidation.js"></script>
<script src="../assets/js/yourfeature.js"></script>
/* 2. Define rules for your form */
var myRules = [
  { field: 'title',  message: 'Title is required.' },
  { field: 'amount', message: 'Enter a positive amount.',
    validator: function(v) { return !isNaN(v) && parseFloat(v) > 0; } }
];

/* 3. Call in your submit handler */
$('#my-submit-btn').on('click', function() {
  if (!GRC.validate('my-form-id', myRules)) return;
  // ... AJAX submission
});
<!-- 4. Add .invalid-feedback after each required input -->
<input type="text" id="title" class="form-control">
<div class="invalid-feedback">Title is required.</div>

<!-- For input-group fields (e.g. password with eye button): -->
<div class="input-group has-validation">
  <input type="password" id="amount" class="form-control">
  <span class="input-group-text">$</span>
</div>
<div class="invalid-feedback">Enter a positive amount.</div>
Server-side validation is untouched. usersaction.php still validates all fields independently. Client-side validation is a convenience layer only — it prevents pointless round-trips but is not a security control.
