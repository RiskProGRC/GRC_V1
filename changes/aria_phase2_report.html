What Phase 2 Does

Phase 2 wires every entity class into the BaseRepository layer built in Phase 1. Each of the 22 classes is updated to extend BaseRepository instead of connectionClass, its boilerplate read methods replaced with single-line calls, all raw SQL replaced with prepared statements, and every inline system_logs INSERT replaced with ActivityLogger::log(). No page files, AJAX wrappers, or the public API of any class changes — callers continue to work without modification.

Updated

BaseRepository.php — Auto-loads ActivityLogger

File: Project/core/BaseRepository.php

One require_once line added so that any entity class that includes BaseRepository automatically gets ActivityLogger without needing a second include.

Before
require_once __DIR__ . '/../connection/connectionClass.php';
After
require_once __DIR__ . '/../connection/connectionClass.php';
require_once __DIR__ . '/ActivityLogger.php';
Impact: Entity classes only need one require_once pointing to BaseRepository — they get the full chain automatically.
Changes Applied to Every Class

Inheritance

  • extends connectionClass → extends BaseRepository
  • require_once → /../core/BaseRepository.php

SQL Safety

  • All real_escape_string() calls removed
  • All string-interpolated queries → prepare() + bind_param()
  • or die($this->error) removed

Audit Logging

  • 3-line INSERT INTO system_logs blocks removed
  • Replaced with ActivityLogger::log($this, $uid, ...)

Read Boilerplate

  • 10-line fetch loops → fetchAll() / fetchWhere()
  • Single-row lookups → fetchOne()
  • Count queries → countAll() / countWhere()
  • Delete → deleteById()

addaction() — Write Method

Before — SQL injection via string interpolation
public function addaction($uid,$ipaddress,$dept_id,$process,$risk,$action,$status,$priority,$timeline){
    $action = $this->real_escape_string($action);

    $insert = "INSERT INTO action(process_id,dept_id,risk_id,action,status,priority,timeline,uid)
               VALUES('$process','$dept_id','$risk','$action','$status','$priority','$timeline','$uid')";
    $query  = $this->query($insert) or die($this->error);
    if(!$query){
        return "DATA NOT ENTERED";
    } else {
        $insert = "INSERT INTO `system_logs`(`user_id`, `entity`, `action`, `ip_address`)
                   VALUES ('$uid','Action','Added Action','$ipaddress')";
        $query  = $this->query($insert) or die($this->error);
        return "Action Added Successfully";
    }
}
After — prepared statement + centralised logging
public function addaction($uid,$ipaddress,$dept_id,$process,$risk,$action,$status,$priority,$timeline){
    $stmt = $this->prepare(
        "INSERT INTO action(process_id,dept_id,risk_id,action,status,priority,timeline,uid)
         VALUES(?,?,?,?,?,?,?,?)"
    );
    $stmt->bind_param('ssssssss', $process,$dept_id,$risk,$action,$status,$priority,$timeline,$uid);
    $stmt->execute();
    if ($stmt->affected_rows < 1) {
        return "DATA NOT ENTERED";
    }
    ActivityLogger::log($this, $uid, 'Action', 'Added Action', $ipaddress);
    return "Action Added Successfully";
}
Refactor

showaction() / dashboard() / pendingaction() — Read Methods

Before — 10-line boilerplate repeated in every class
public function showaction(){
    $select = "SELECT * FROM action";
    $result = $this->query($select) or die($this->error);
    $count  = $result->num_rows;
    if($count < 0){ return "NO RECORDS FOUND"; }
    else {
        while($row = $result->fetch_assoc()){ $rows[] = $row; }
        if(!empty($rows)){ return $rows; } else { return array(); }
    }
}
public function dashboard(){
    $select = "SELECT * FROM action";
    $query  = $this->query($select) or die($this->error);
    $count  = $query->num_rows;
    return $count;
}
public function pendingaction(){
    $select = "SELECT * FROM action where approval='1'";
    $query  = $this->query($select) or die($this->error);
    $count  = $query->num_rows;
    return $count;
}
After — delegated to BaseRepository helpers
public function showaction(): array {
    return $this->fetchAll('action');
}
public function dashboard(): int {
    return $this->countAll('action');
}
public function pendingaction(): int {
    return $this->countWhere('action', 'approval', '1');
}

Excluded from Phase 2

loginClass — Deferred to Dedicated Auth Sprint

File: Project/login/loginClass.php

loginClass handles password verification, session assignment, Google OAuth registration, password reset token generation, and login attempt tracking. Modifying it as part of a structural migration carries disproportionate risk — a regression here locks every user out of the system. It remains on connectionClass and is flagged for a focused auth-security sprint after Phase 2 is validated in production.

Risk if deferred: loginClass still uses real_escape_string() on user input. Acceptable short-term; must be addressed before public deployment.
